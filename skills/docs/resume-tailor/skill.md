# Doug's Resume Tailoring — Complete Honest Adaptation Guide

## User Context

Doug is a QA & Process Engineer transitioning into AI. He has **two target profiles**:

| Profile | Target Roles | Relevant Stack |
|---|---|---|
| **Master A** | QA / Process Engineering (current) | Playwright, TypeScript, Robot Framework, BrowserStack, GitLab CI/CD, GitHub Actions, BDD, Scrum |
| **Master B** | AI Quality / Evaluation / MLOps (transition) | Playwright as evaluation pipeline, CI/CD as MLOps, BDD as behavioral testing, FIAP AI degree, data analysis |

---

## Core Philosophy

> Tailoring your resume is not about pretending to be something you're not.
> It's about **shining a light on the right angle of who you genuinely are** — for the right recruiter, at the right moment.

The ethical line is clear:

| ✅ Honest Tailoring | ❌ Lying (never do this) |
|---|---|
| Reordering bullets to highlight the most relevant | Fabricating a job, company, or project |
| Using the job's exact language to describe what you did | Inflating proficiency in something you don't master |
| Omitting tasks irrelevant to the role | Making up freelance clients or projects |
| Honest reframing of transferable experience | Claiming a degree not yet completed |
| Adjusting the summary angle per role | Using a title higher than your actual one |
| Listing FIAP as "in progress (2025–2027)" | Omitting that it's still in progress |

**Golden rule:** If you can't defend a claim in an interview with real examples, it doesn't belong on your resume.

## STEP-BY-STEP — The Tailoring Ritual

### STEP 1 — Analyze the Job Posting

Before editing anything, extract from the posting:

1. **Required keywords** — skills and tools listed as "required" or "must have"
2. **Preferred keywords** — listed as "nice to have" or "preferred"
3. **Company language** — exact terms they use (e.g., "test automation" vs "automated testing")
4. **Primary focus** — which responsibility appears first or most frequently?
5. **Explicit soft skills** — "cross-functional collaboration", "ownership", "attention to detail", etc.

> 💡 Tip: Words that appear more than once in a posting are almost certainly ATS keywords.

---

### STEP 2 — Choose the Right Master Resume

| If the role is... | Use Master |
|---|---|
| QA Engineer, SDET, Test Automation, Process Engineer | **Master A** |
| AI Quality Engineer, AI Evaluation, Red Teaming, Safety Engineer | **Master B** |
| MLOps, DevOps/AI, Platform Engineer | **Master B** (emphasize CI/CD) |
| Hybrid QA+AI / "QA for ML systems" | **Master B** with elements from A |

---

### STEP 3 — Rewrite the Professional Summary

The summary is **always** customized. It's the first thing a recruiter reads.

**Ideal structure (3–4 lines):**
```
[Title aligned to the role] with [X years] of experience in [key area from posting].
Track record of [relevant achievement] with focus on [company's priority].
Currently [relevant transition context or specialization].
```

**Example — AI Quality Engineer posting:**
> QA Engineer transitioning into AI Quality with 5+ years building automated testing pipelines (Playwright/TypeScript) and designing behavioral acceptance criteria for product systems. Track record of data-driven analysis correlating system changes with business outcomes. Currently pursuing B.Tech in Artificial Intelligence at FIAP (2025–2027).

**Example — Senior QA Engineer posting:**
> QA & Process Engineer with 5+ years in automation, CI/CD integration, and cross-functional quality strategy for SaaS and lead generation products. Proven track record building Playwright/TypeScript test suites from scratch and reducing release cycle defects through BDD-driven acceptance criteria.

---

### STEP 4 — Reorder and Reframe Experience Bullets

**Rule:** The first 2–3 bullets under each role should be the most relevant to the posting. Don't invent — reorder and reframe.

**Honest Reframing Principle:**
Same experience, different angle — depending on where you're going.

| Real Experience | Reframe for QA/SDET | Reframe for AI Quality / MLOps |
|---|---|---|
| Wrote test cases with Playwright/TypeScript | "Built end-to-end test automation suite using Playwright/TypeScript, reducing regression cycle by X%" | "Designed automated behavioral evaluation pipelines using Playwright/TypeScript for product quality assurance" |
| CI/CD with GitHub Actions | "Integrated automated test execution into GitHub Actions CI/CD pipeline" | "Implemented deployment validation pipelines via GitHub Actions — foundational MLOps pattern" |
| Correlation between deploys and call volume | "Conducted root cause analysis correlating code deployments with conversion funnel performance" | "Applied data-driven correlation analysis to measure impact of system changes on business outcomes" |
| BDD/Gherkin scenarios | "Authored BDD test scenarios in Gherkin for feature acceptance across multiple product verticals" | "Defined behavioral contracts and acceptance criteria for AI-adjacent product features using BDD methodology" |
| iOS/BrowserStack compatibility testing | "Led cross-browser/cross-device QA using BrowserStack, identifying iOS 14.x as root cause of layout regressions" | "Conducted systematic regression analysis across device matrix, identifying root causes through structured testing methodology" |
| Call/form performance analysis for CEO | "Produced executive-level bilingual quality reports mapping test findings to business KPIs for C-suite" | "Translated technical quality findings into business impact analyses for executive decision-making" |

---

### STEP 5 — Adjust the Skills Section

**Rule:** Mirror the exact terms from the job posting. ATS doesn't know "automated testing" = "test automation".

- Use the posting's terms when they describe something you genuinely do
- Don't list skills you can't demonstrate in an interview
- For skills currently being learned: `Python (learning)` or `LLM Evaluation (academic, FIAP)`

**Recommended order for Master B (AI):**
1. Skills listed as "required" in the posting → first
2. Skills listed as "preferred" → second
3. Relevant transferable skills → third

---

### STEP 6 — Remove the Noise

Less is more. For each item, ask: **"Does this bullet help the recruiter see me as the right person for THIS role?"**

- If no → remove or move down
- Omitting irrelevant experience is not lying — it's strategic focus

---

### STEP 7 — ATS Check and Save

**Recommended free tools:**
- **Jobscan** (jobscan.co) — paste job description + resume → ATS compatibility score
- **Teal HQ** (tealhq.com) — integrated keyword finder + match scoring
- **Rezi** (rezi.ai) — scoring across 23 ATS metrics

**Pre-submission checklist:**
- [ ] Summary mentions the job title or a close equivalent?
- [ ] Required keywords from the posting are present in the resume?
- [ ] First bullets under each role are the most relevant?
- [ ] No skills listed that can't be defended in an interview?
- [ ] FIAP degree marked as "in progress" or "expected 2027"?
- [ ] File saved with a descriptive name? (e.g., `Resume_Doug_Anthropic_AIQuality_2025.pdf`)

## The 10 Golden Rules

1. **One master resume per profile** — don't try to cover QA and AI with the same document
2. **Summary is always customized** — it's the first thing read; it's the most important
3. **Reorder, don't invent** — move the most relevant bullets to the top
4. **Mirror the job's language** — ATS and recruiters search for their own terms
5. **Omitting ≠ lying** — removing the irrelevant is strategy, not dishonesty
6. **Reframing is honest** — the same fact can be described from different angles; the fact must be true
7. **Real metrics only** — if you don't have the exact number, use "~X%" or omit it
8. **FIAP always as "in progress"** — never as a completed degree
9. **Save with company/role name** — enables version control and follow-up tracking
10. **Run an ATS check before submitting** — Jobscan or Teal, free, 5 minutes
