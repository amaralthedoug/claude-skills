# Resume Format, Layout & ATS Optimization — Master Guide

> Synthesized from 30 sources: Harvard FAS, Oxford Careers, TheLadders Eye-Tracking
> Study, Jobscan, Teal HQ, Indeed, Enhancv, The Interview Guys, Fast Company,
> Microsoft Word, CandyCV, Resumly, ResuFit, Vocationic, and more.

## The Two Audiences — Never Forget This

Every resume formatting decision must satisfy two audiences **simultaneously**:

1. **The ATS (bot)** — parses your text, extracts data, scores keyword match
2. **The Human (recruiter)** — scans visually for ~6–10 seconds before deciding

A formatting choice that impresses humans but confuses the ATS = invisible resume.
A formatting choice that passes ATS but is ugly/unreadable to humans = rejected resume.
**The goal is always both.**

## SECTION 1 — FILE FORMAT

### What to submit

| Format | Verdict | Why |
|---|---|---|
| `.docx` | ✅ Gold standard | Parsed correctly by virtually every ATS on the market |
| `.pdf` (text-based) | ✅ Safe if exported from Word/Google Docs | Preserves formatting; most modern ATS handle it well |
| `.pdf` (scanned/image) | ❌ Never | Completely unreadable to ATS — treated as a blank image |
| `.rtf` | ❌ Avoid | Inconsistent parsing across ATS systems |
| `.png` / `.jpg` | ❌ Never | Image files are unreadable to ATS |

**Best practice:** Submit `.docx` unless the posting explicitly requests PDF. If PDF, export from Word — never scan or screenshot.

**Self-test:** Paste your entire resume into Notepad. If the plain text is readable and in the correct order, your file will parse cleanly in ATS. If sections are jumbled, simplify your formatting.

## SECTION 2 — LAYOUT & STRUCTURE

### Column Layout

| Layout | ATS Compatibility | Best For |
|---|---|---|
| **Single column** | ✅ Best | All industries; always ATS-safe |
| **Two columns** | ⚠️ Risky | Creative roles only; ATS often misreads or skips the right column |
| **Three+ columns** | ❌ Avoid | ATS parses them as gibberish |
| **Sidebars** | ❌ Avoid | ATS cannot interpret sidebar text correctly |

**Rule:** When in doubt, single column. Up to 88% of resumes with visual/column layouts are discarded by certain ATS filters.

### Section Order (Reverse Chronological — Preferred by 90% of recruiters)

For **Doug's QA profile (Master A):**
```
1. Name + Contact Info
2. Professional Summary
3. Skills
4. Work Experience (reverse chronological)
5. Education
6. Certifications
```

For **Doug's AI/Transition profile (Master B):**
```
1. Name + Contact Info
2. Professional Summary
3. Skills (AI-relevant front and center)
4. Work Experience (reverse chronological)
5. Education (FIAP first — most relevant)
6. Certifications / Courses
7. Projects (if applicable)
```

### Section Headers — Use Standard Names

ATS software categorizes sections by header text. Non-standard names cause misclassification.

| ✅ Use This | ❌ Not This |
|---|---|
| Work Experience | Where I've Made an Impact / Career Journey |
| Skills | My Toolkit / Competencies |
| Education | Academic Background |
| Summary | About Me |
| Certifications | Credentials |

### What to Never Include in Headers or Footers

ATS systems frequently skip headers and footers entirely. Place all critical information in the **main body**:
- Your name
- Phone number
- Email address
- LinkedIn URL
- Location

### Contact Info Format

```
Douglas Rafael do Amaral
Joinville, SC, Brazil  |  +55 (XX) XXXXX-XXXX  |  email@email.com  |  linkedin.com/in/yourhandle
```

City + State/Country only — no full street address needed.

## SECTION 3 — TYPOGRAPHY & FONTS

### ATS-Safe Font List (Use Only These)

**Sans-serif (modern, screen-optimized — recommended):**
- **Calibri** — corporate gold standard; Microsoft default; clean and ATS-safe
- **Arial** — universally safe; neutral; great for tech roles
- **Helvetica** — polished; slightly more editorial
- **Lato** — modern; excellent screen readability
- **Source Sans Pro** — clean; great for AI/tech profiles
- **Tahoma** — clear; ATS-safe
- **Verdana** — wide; highly readable on screens

**Serif (traditional, professional — use sparingly):**
- **Cambria** — designed for screen readability in small sizes; ATS-safe
- **Georgia** — clean serif; good for leadership/consulting roles
- **Garamond** — elegant; ATS-safe but avoid for tech roles
- **Times New Roman** — universally recognized; ATS-safe but dated

**Never use:**
- Comic Sans, Papyrus, Brush Script, Courier, any decorative/display fonts
- Custom or downloaded fonts not in the standard OS library
- Fonts with unusual glyphs or ligatures — ATS may misread individual characters

**Best practice for Doug:** Use **Calibri** (body) throughout. Clean, ATS-safe, and widely accepted in both tech and QA roles.

### Font Sizes

| Element | Size |
|---|---|
| Your name (header) | 16–20 pt |
| Section headers | 12–14 pt |
| Job titles | 11–12 pt |
| Body text / bullets | 10–11 pt |
| Company names, dates | 10–11 pt |

**Minimum:** Never go below 10 pt anywhere. Below that, ATS may misread and humans can't scan quickly.

### Font Colors

- **Primary:** Black (`#000000`) or near-black (`#1a1a1a`) for all body text
- **Secondary accent:** Dark navy (`#1a3a5c`) or dark grey (`#555555`) — for section headers only
- **Rule:** Maximum 2 colors. More than 2 = unprofessional and ATS risk
- **Never:** Red, green, orange, or light colors — low contrast causes parsing errors

### Bold, Italic, Underline

| Formatting | When to Use | ATS Risk |
|---|---|---|
| **Bold** | Job titles, company names, key achievements | Safe — use selectively |
| *Italic* | Dates, supplementary info | Safe — but avoid for critical info |
| Underline | Avoid entirely | Can be misread by ATS as hyperlinks |
| ALL CAPS | Section headers only (optional) | Safe if used consistently |

## SECTION 4 — SPACING & MARGINS

### Margins

| Setting | Value | Notes |
|---|---|---|
| Standard | **1 inch (2.54 cm) all sides** | Default; ATS-safe; preferred by 90% of recruiters |
| Minimum | **0.5 inch (1.27 cm)** | Never go below this — ATS misreads text near page edges |
| Reduce to fit | **0.7–0.75 inch** | Only when content genuinely doesn't fit at 1 inch |

**Rule:** If you need margins below 0.7 inches to fit content — the problem is content volume, not margins. Cut or go to page 2.

### Line Spacing

| Context | Spacing |
|---|---|
| Body text (bullets) | **1.0 – 1.15** |
| Between bullet points | **0.2 – 0.5 pt gap** |
| Between sections | **8–12 pt space after** (or a blank line) |
| Section headers | **1.15 – 1.4** line height |
| Name / contact block | **1.0** |

**ATS rule:** Single or 1.15 line spacing is the safe zone. Extra blank lines or excessive padding waste space and can confuse ATS parsers.

### White Space — The Most Undervalued Element

White space (negative space) is not wasted space. It:
- Improves readability by ~20% (ResuFit / Indeed research)
- Naturally guides the recruiter's eye to critical sections
- Prevents cognitive overload during the 6–10 second scan
- Helps ATS parse sections cleanly without overlap

**Ideal content-to-whitespace ratio:** approximately 60–70% content, 30–40% white space.

**How to create white space without wasting it:**
- Space before section headers (not after each bullet)
- Consistent indentation for bullet points
- One blank line between roles within the same section
- No double blank lines anywhere

## SECTION 5 — VISUAL HIERARCHY

### The F-Pattern (Eye-Tracking Research — TheLadders / Fast Company)

Recruiters scan in an **F-pattern**:
1. Horizontal sweep across the **top** (name, title, contact)
2. Horizontal sweep across the **summary / first section**
3. Vertical scan down the **left margin** (company names, dates, section headers)
4. Occasional short horizontal glances into the right for details

**Implication:** Place your strongest content **top-left**. The top third of page 1 and the left margin are premium real estate.

### Visual Hierarchy Rules

- **Size contrast:** Section headers larger than body text (12–14 pt vs 10–11 pt)
- **Weight contrast:** Bold for job titles and company names; regular for descriptions
- **Position:** Most relevant experience and skills at the top
- **Whitespace as separator:** Between sections — not between every bullet

### The 4 Elements Recruiters See First (80% of initial scan time)

According to eye-tracking research, recruiters focus 80% of their initial seconds on:
1. **Your name**
2. **Current/most recent job title**
3. **Current/most recent company**
4. **Education**

Design your resume so these four are instantly visible without scanning.

## SECTION 6 — ELEMENTS TO NEVER USE

The following elements break ATS parsing and should **never** appear in a resume submitted digitally:

| Element | Why It Breaks ATS |
|---|---|
| Tables | ATS reads table cells out of order or skips them |
| Text boxes | Content inside is often invisible to ATS parsers |
| Graphics / icons / charts | ATS cannot extract text from images |
| Skill bars (visual) | "80% Python" means nothing to ATS — use text only |
| Columns / sidebars | ATS reads left-to-right, misplacing sidebar content |
| Headers / footers | Frequently skipped entirely by ATS |
| Photos / headshots | Not used in US/Brazil applications; creates bias risk |
| Logos | Images; ATS cannot parse them |
| Horizontal lines (decorative) | Can break section parsing in some ATS |
| Special symbols (★ ➤ ☑) | May render as garbled characters in ATS databases |
| Fancy bullet symbols | Use standard round bullets (•) or hyphens (-) only |
| Justified text alignment | Creates irregular word spacing; harder to read and parse |

**Safe alternatives:**
- Replace tables → plain text with consistent spacing
- Replace skill bars → "Python (advanced)", "Playwright (expert)"
- Replace graphics → quantified bullet points
- Replace icons → plain text labels

## SECTION 7 — RESUME LENGTH

| Career Stage | Recommended Length |
|---|---|
| Entry level (0–3 years) | 1 page |
| Mid-level (3–10 years) | 1–2 pages |
| Senior / 10+ years | 2 pages max |
| Academic / Research CV | 3+ pages acceptable |

**For Doug:** 1–2 pages depending on role. AI/Evaluation roles may benefit from a Projects section that justifies page 2.

**If you're squeezing to fit one page:**
- Reduce margins to 0.75 inch (not below 0.7)
- Reduce body font to 10 pt (not below 10)
- Remove irrelevant bullets (better quality, fewer lines)
- Remove pre-2015 experience if not directly relevant

## SECTION 8 — BULLET POINTS

### Format
- Use **standard round bullets** (•) — always ATS-safe
- Hyphens (-) are also acceptable
- **Never use:** ✓ ★ ➤ ❯ or any special character bullets

### Structure of a Strong Bullet (STAR-lite formula)
```
[Action Verb] + [What You Did] + [How / With What] + [Result / Impact]
```

Example:
> Built end-to-end test automation suite using Playwright/TypeScript, reducing manual regression cycle by 40% and enabling weekly release cadence.

### Rules
- Start every bullet with a **strong action verb** (never "Responsible for" or "Helped")
- Include **quantified metrics** wherever possible: %, time saved, volume, scale
- Keep each bullet to **1–2 lines** maximum
- 3–6 bullets per role is the sweet spot
- Most relevant bullet **always goes first** within each role

## SECTION 9 — ATS KEYWORD STRATEGY

### Where to Place Keywords
1. **Professional Summary** — 2–3 critical keywords naturally embedded
2. **Skills Section** — exact terms from the job posting
3. **Work Experience bullets** — demonstrate keywords in context (not just list them)
4. **Job Titles** — if your real title differs, add the standard title in parentheses

### How to Find the Right Keywords
- Words that appear **more than once** in a job posting → high priority ATS keywords
- Skills listed under "Required" → mandatory
- Skills listed under "Preferred" → include if you have them
- Mirror exact phrasing: if they write "test automation", don't write "automated testing"

### Keyword Frequency
- Target keywords should appear **2–3 times** across the resume (summary + skills + experience)
- More than 3–4 uses of the same term = keyword stuffing → flagged by modern AI-powered ATS

### Spell Out Acronyms Once
- Write both versions: "Applicant Tracking System (ATS)", "Continuous Integration (CI)"
- This covers both full-text and abbreviated keyword searches

### Skills Section Format (ATS-optimized)
```
Test Automation | Playwright | TypeScript | GitHub Actions | CI/CD
BDD / Gherkin | Robot Framework | BrowserStack | Agile / Scrum
LLM Evaluation (academic) | Python (learning) | Supabase
```

Use `|` or commas as separators. Never use a table or columns for skills.

  □ Line spacing: 1.0–1.15 for body
